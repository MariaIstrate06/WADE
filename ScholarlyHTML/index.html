TRASD - Traffic Sign Smart Detector

Authors
Cristi Rusu & Denis Aenășoaei & Maria Istrate

Abstract

TRASD is a (micro-)service based Web System able to detect road/traffic signs marking a highly frequented route (by vehicles or pedestrians) given by the user in the form of a video or a picture. For each recognized road sign, information regarding regulatory aspects, type, meaning and relationships to other signs is provided by the system, adding suggestions regarding user behaviour.

Introduction

There are many applications out there that might offer information regarding road sign positioning by accessing map application APIs, yet what TRASD brings to the table is a micro-service based system that collects every sign and gives explanations and other regulatory information to the user.

Since the modern ages, GPS systems have proven their necessity and people have become more and more dependent on them as they can manage to ease the trouble of travelling. Besides that, out-of-the-country travelling without having to ask for direction is now easier than ever. But wouldn't it be easier if the signs shown on the maps also had explanations, taking into consideration that their looks may differ from country to country.

Goals:

  • Create and develop a useful micro-service based web system.
  • Be fully funcitioning on modern Web browsers
  • Be customizable for inclusion in arbitrary Web sites, while remaining easy to process and interoperable.
  • Build on top of open, royalty-free standards.
  • Long-term viability as a data format.
  • Proper use of ontologies.
  • Present everything in a easy-to-read and embed format.

Structure

We will describe below the modules and components that we use for our project.

3.1 Image processing

Video Recognition and Image Processing using Python, OpenCV and Tensorflow.

3.2 BackOffice API

A SPARQL Query Service is an HTTP Service (also known as a Web Service) that offers an API for performing declarative Data Definition and Data Manipulation operations on data represented as RDF sentence collections.

3.3 Frontend Client

Angular 10 Single Page App containing a Data Presenter, Upload Service and API Caller.

3.3 Crawling Services

  • Waze
  • Google Maps
  • Road Sign DB
